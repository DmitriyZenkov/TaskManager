Комментарии:
Чтобы проект запустился нужно изменить два connectionString в файле WebConfig. DefaultConnection и GroupTasksEntities. У меня локально все запускается без проблем. Еще нужно создать базу данных MSSQL с использованием приложенного скрипта (grouptasks db script.sql).

Начальные данные в проект добавляются автоматически при первом запуске:
3 роли: супер администратор, администратор проекта, пользователь (работник);
10 проектов;
50 задач.

Логины и пароли приложены отдельным txt файлом (Логины.txt).

Нормализация БД.
У таблицы AspNetUserRoles два ключевых поля потому что Visual Studio сама сгенерировала их когда я создавал проект. В проекте используется два dbContext, один сгенерированный при создании проекта и второй после добавления ADO.NET Entity Data Model с помощью Entity Framework.
БД напрямую желательно не редактировать, только через GUI.

На главной страницы супер администратора стоит фильтр задач. Чтобы там появились задачи нужно их добавить с помощью GUI. Нужно выставить соответствующие даты, чтобы они подходили под условие фильтра. Нужно изменить статус задач.

К сожалению не успел сделать представления по собственным задачам с группировкой по проекту, работнику, статусу для каждой группы пользователей.
Так же не сделал Unit Tests потому что не знаю как и не хватило времени разобраться с этим.

Это мое первое приложение на платформе ASP.NET с шаблоном MVC. Сильные стороны: выучил основы быстро и добился результата. Слабые стороны: мало опыта с MVC.
